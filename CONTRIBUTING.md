# Contributing to StadiumPilot AI

First off, thank you for considering contributing to StadiumPilot AI! It's people like you that make open-source projects such a great community.

## 1. Where do I go from here?

If you've noticed a bug or have a feature request, please make sure to check if there is an existing issue that addresses it. If not, feel free to open a new issue!

## 2. Fork & create a branch

If this is something you think you can fix, then fork StadiumPilot AI and create a branch with a descriptive name.

## 3. Get the test suite running

Make sure your environment is configured correctly. Run `npm install` in both the `frontend` and `backend` directories. Ensure that `npm run test` passes on the backend, and that `npm run lint` passes without warnings on the frontend.

## 4. Implement your fix or feature

At this point, you're ready to make your changes. Feel free to ask for help if you get stuck.

## 5. Make a Pull Request

Once your changes are ready, push your branch and open a Pull Request against the `main` branch. Please provide a clear description of the problem you're solving and the changes you've made. 

A maintainer will review your code and may suggest some changes before merging.
